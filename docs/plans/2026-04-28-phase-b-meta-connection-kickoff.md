## What Phase B unlocks

Phase B is the first slice that actually publishes to Meta. After Phase B, the
owner clicks "Post to Facebook" inside `/admin` and the post appears on the
Facebook Page within seconds — no copy/paste, no manual posting.

**Phase B scope**:

- Add a Supabase Edge Function `meta-publish` that:
  - Accepts `{ watchId, platform: 'facebook' }` from an admin caller.
  - Validates the caller is on the admin allowlist.
  - Reads the saved draft caption + media URL from `watch_alley.social_posts`.
  - Calls the Meta Graph API to publish to The Watch Alley Facebook Page.
  - Stores the resulting external post id, URL, and timestamp back into
    `watch_alley.social_posts`, flipping `status` from `draft` to `published`.
- Add `Post to Facebook` button inside the admin Social Publishing panel,
  wired to the new Edge Function.
- Show the published Facebook post URL in the admin once available, with a
  "View on Facebook" link.
- Prevent duplicate accidental reposts (existing `unique (watch_id, platform)`
  + the on-conflict guard that already preserves terminal `published` state).
- Surface retryable error state if Meta rejects the post.

Phase C follows for Instagram (different flow: needs a media container + poll
for status before publish).
